Paradigma Orientado a Objeto (POO)

Historia da Programação Orientada a Objeto (POO)

  • Conceitos Principais de POO: O paradigma orientado a objetos representa o mundo real através de objetos que interagem.
  • Classes em POO: Classes são moldes para objetos, sendo um objeto uma instância de uma classe.
  • Atributos: Características de um objeto que armazenam valores e determinam seu estado.
  • Métodos: Ações executadas pelo objeto, engatilhando estímulos e possíveis mudanças.
  • Superclasse e Subclasse: Classes podem compartilhar características, com subclasses herdando propriedades de superclasses.
  • Abstração: Importante para filtrar informações relevantes ao criar classes.
  • Encapsulamento: Permite que objetos funcionem sem acessar diretamente seu interior.
  • Herança: Permite que subclasses herdem características de superclasses, evitando repetição.
  • Polimorfismo: Um método pode ser aplicado de formas diferentes em várias classes.
  • POO vs Programação Estruturada: POO foca em componentes e suas interações, enquanto programação estruturada divide problemas em partes menores.

Historia do Dart

  • - Dart (originalmente denominada Dash) é uma linguagem de script voltada à web desenvolvida pela Google através dos engenheiros Lars Bak (Google Chrome) e Kasper Lund. Ela foi lançada na GOTO Conference 2011, que aconteceu de 10 a 11 de outubro de 2011 em Aarhus, na Dinamarca.
  • - Para poder ter o seu lançamento oficializado, foi necessária a aprovação do comitê técnico TC52 Ecma International, onde a linguagem conseguiu provar que possui todos os padrões de desenvolvimento e é aceita em todos os navegadores modernos.
  • - Em fevereiro de 2018, o Dart sofreu o primeiro reboot e assim foi lançado o Dart 2, otimizado para desenvolvimento no Client-Side para a web e incluída para o mundo mobile.
  • - Atualmente o Dart possui 2 soluções: o Dart Native e o Dart Web.
  • - O Dart Native é utilizado para o desenvolvimento mobile, desktop e servidores. Ele inclui a máquina virtual do Dart com a copilação JIT (just-in-time) e o copilador AOT (ahead-of-time) que produz os códigos de máquina.
  • - O Dart Web tem como foco o desenvolvimento web, como o nome diz, e traz o Dartdevc que compila o código para desenvolvimento e o dart2js que é o compilador indicado para fazer a build para produção.
  • - Para o desenvolvimento web, server ou script de comando, é necessária a utilização do Dart SDK, que é uma biblioteca e uma ferramenta de linha de comando. Se o objetivo for o desenvolvimento mobile, o SDK não é necessário, basta fazer a instalação do Flutter.
  • - O Dart tem como principal característica o fato de ser fortemente tipada, porém a declaração dos tipos pode ser explícita ou não, ou seja, a própria linguagem pode inferir o tipo da variável.
  • - Como principais frameworks da linguagem temos:
    • Flutter
    • AngularDart
    • Material Design Lite
    • OverReact
    • VueDart
    • Aqueduct
    • Angel
    • Jaguar

Atributos da Programação Orientada a Objeto (POO)

  • Conceitos Principais de POO: Paradigma orientado a objetos como representação do mundo real através de componentes interagindo entre si.
  • Classes em POO: Classes são moldes para objetos que merecem representação no sistema; objetos são instâncias de classes.
  • Atributos: Características de um objeto, acessíveis apenas por métodos da classe.
  • Métodos: Ações executadas pelo objeto que podem gerar mudanças nele.
  • Superclasse e Subclasse: Superclasse compartilha características com subclasses que herdam essas características.
  • Abstração: Foco em filtrar características essenciais para o sistema.
  • Encapsulamento: Objetos interagem entre si sem acessar diretamente seu interior, apenas por métodos.
  • Herança: Subclasses herdam características da superclasse, evitando duplicação de código.
  • Polimorfismo: Um mesmo método pode ser aplicado de formas diferentes em diversas classes.
  • POO vs Programação Estruturada: POO foca nos componentes e interações, enquanto a programação estruturada divide o problema em partes menores.

Atributos da Linguagem Dart

  • Sintaxe: Baseada na linguagem C, a sintaxe do Dart é clara e concisa.
  • Tipagem: O Dart é uma linguagem de tipagem estática, o que significa que o tipo de dado de uma variável deve ser declarado explicitamente.
  • Orientação a objetos: O Dart suporta a programação orientada a objetos, trazendo vantagens como reutilização, legibilidade e manutenção do código.
  • Compilação: O Dart pode ser compilado em código nativo ou JavaScript.
  • Execução: O Dart pode ser executado em ambientes nativos e web.
  • Recursos: Possui uma variedade de recursos, como interfaces, classes abstratas e coleções.
  • Velocidade: Permite criação rápida de aplicações devido aos recursos JIT e AOT (just-in-time e ahead-of-time).
  • Análises estáticas: Comporta análises estáticas para detectar erros.
  • Suporte a JavaScript: Em aplicações web, oferece suporte a JavaScript, bibliotecas, HTML e CSS.

Curiosidades sobre o Dart

  • Dart foi criada pela Google e lançada em 2011 na Conferência GOTO, na Dinamarca.
  • Seus criadores foram Lars Bak, contribuidor do Google Chrome, e Kasper Lund, engenheiro de software.
  • O objetivo inicial da Dart era substituir o JavaScript como a linguagem principal dos navegadores.
  • É uma linguagem multiplataforma, permitindo que o código seja escrito uma vez e executado em diversas plataformas.
  • Dart é de tipagem estática, exigindo declaração explícita do tipo de dado de uma variável.
  • É otimizada para interface de usuário.
  • Dart é baseada em compilações de códigos JavaScript e pode ser compilada em código nativo ou JavaScript.
  • Dart é orientada a objetos e multiparadigmas.
  • Possui uma sintaxe baseada na linguagem C.
  • Dart é mais rápida que o JavaScript.
